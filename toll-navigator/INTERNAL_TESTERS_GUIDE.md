# Добавление тестеров в Internal Testing

## Зачем
Без тестеров Internal Testing релиз не достигнет ни одного пользователя. Только добавленные email получат доступ к app через Play Store.

## Пошагово

### Шаг 1: Открыть раздел тестеров
1. Google Play Console → **HaulWallet**
2. Слева: **Test and release → Internal testing**
3. Вверху вкладки: **"Testers"** (или "Тестеры")

### Шаг 2: Создать список тестеров
1. Нажми **"Create email list"** (или "Create list")
2. **List name:** `HaulWallet Team`
3. **Add testers:** вставь email (свой или тех кого хочешь пригласить)
   - Пример: `vladimir92905@gmail.com`
   - Можно несколько через запятую
4. Нажми **"Save"**

### Шаг 3: Включить тестеров для релиза
1. Вернись во вкладку **"Releases"**
2. Открой текущий draft релиз
3. В разделе **"Testers"** выбери созданный список `HaulWallet Team`
4. Нажми **"Save"**

### Шаг 4: Приглашение
- Каждый тестер получит email с ссылкой
- Переходит по ссылке → становится тестером
- Открывает Play Store → видит HaulWallet в разделе "Тестирование приложений"
- Может установить как обычное приложение

## Лимиты
- **До 100 тестеров** на Internal Testing
- Тестеры не видят app в обычном поиске Play Store
- Можно добавить/удалить в любой момент

## Что дальше (после API 35 fix)
1. ✅ Добавить тестеров
2. ✅ Загрузить новый AAB (version code 4)
3. ✅ Review release → **Start rollout to Internal testing**
4. Получить приглашение → установить → протестировать
5. Если всё ок → Promote to **Closed testing** → потом **Production**
