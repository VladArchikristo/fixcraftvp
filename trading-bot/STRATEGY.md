## 🔒 VALIDATION LAYER — 4 ПРАВИЛА (нарушение = блок)

### Правило 1: Минимальный консенсус
- **Минимум 2 реальных стратегии** должны давать одинаковый сигнал
- Считается: FUNDING_EXTREME, OI_DIVERGENCE, WHALE_WALLS, VAULT_COPY, MTF_CONFLUENCE
- НЕ считается: ADX_FILTER (вспомогательный)
- `real_count < 2` → **БЛОК**

### Правило 2: ADX Блокировка
- Если ADX > 30 И сигнал против тренда → **ПОЛНЫЙ БЛОК** (не штраф!)
- trend_dir = LONG (plus_di > minus_di), trend_dir = SHORT (minus_di > plus_di)
- Пример: тренд вверх (ADX=35), сигнал SHORT → **БЛОК**

### Правило 3: StochRSI Фильтр
- LONG при StochRSI k > 80 (перекупленность) → **БЛОК**
- SHORT при StochRSI k < 20 (перепроданность) → **БЛОК**

### Правило 4: Net Score порог
- `abs(net_score) < 25` → сигнал слабый → **NEUTRAL**, не торгуем

## ⛔ ЗАПРЕТЫ (никогда не делать)

1. **Не входить на одном сигнале** — нужен консенсус 2+ стратегий
2. **Не торговать против сильного тренда** (ADX > 30)
3. **Не шортить при перепроданности** (StochRSI < 20)
4. **Не лонговать при перекупленности** (StochRSI > 80)
5. **Не превышать дневной лимит убытков**
6. **Не открывать 4-ю позицию** при уже 3 открытых
7. **Не торговать в paper mode** с реальными деньгами
8. **Не игнорировать emergency stop**

## 📋 ОРДЕР-ЧЕКЛИСТ (перед каждой сделкой)

# Псевдокод проверки перед ордером
signals = combine_strategies(...)

if not validated:
    return "NO TRADE — validation failed"

if best['net_score'] < 25:
    return "NO TRADE — weak signal"

ok, reason = risk_manager.check_order(size_usd, balance, open_count)
if not ok:
    return f"NO TRADE — risk: {reason}"

# Всё прошло → открываем
place_order(best['coin'], best['direction'], ...)
